Protocolo de Red, Servidor y Cliente

IPv4 vs IPv6

IPv4

  • Es la cuarta versión del Protocolo de Internet, lanzada en 1981, y sigue siendo el protocolo más usado para la identificación y comunicación de dispositivos en redes.
  • Usa direcciones de 32 bits, permitiendo alrededor de 4.294 millones de direcciones únicas.
  • Formato: cuatro octetos decimales separados por puntos. Ejemplo:
    192.168.1.1
  • Es un protocolo sin conexión, basado en el modelo de entrega “best-effort” (no garantiza la entrega).
  • amiento de direcciones, falta de soporte nativo para IPsec, encabezados limitados y costos crecientes por dirección.

IPv6

  • Es la evolución de IPv4, diseñado para resolver el agotamiento de direcciones y mejorar la eficiencia de la red.
  • Usa direcciones de 128 bits, permitiendo un número prácticamente ilimitado de direcciones.
  • Formato: hexadecimal, separado por dos puntos. Ejemplo:
    2001:0db8:85a3:0000:0000:8a2e:0370:7334
  • Características: configuración automática simplificada, soporte nativo para IPsec, calidad de servicio (QoS) integrada, eliminación de NAT y mejor manejo de multidifusión.
  • Tipos de transmisión: unicast, anycast y multicast.

Servidores

Windows Server

  • Sistema operativo de Microsoft para gestionar redes, datos y aplicaciones empresariales.
  • Funcionalidades: controlador de dominio, servidor de archivos, DNS, DHCP, virtualización, administración centralizada y seguridad avanzada.
  • Plataforma principal para Active Directory y servicios de red en entornos empresariales.

Linux Server

  • Sistema operativo de código abierto, muy usado para servidores web, bases de datos, correo y aplicaciones empresariales.
  • Ventajas: estabilidad, seguridad, personalización, bajo costo y gran variedad de aplicaciones.
  • Usos: hosting web, colaboración, gestión de proyectos, VoIP, streaming, redes sociales privadas, entre otros.

macOS Server

  • Menos común en entornos empresariales, pero útil para servicios de archivos, colaboración y pequeñas redes.
  • Integración con hardware Apple y facilidad de uso.

Servicios de Red

Active Directory

  • Servicio de directorio de Microsoft para gestionar usuarios, permisos y recursos de red.
  • Organiza objetos (usuarios, grupos, dispositivos) en una estructura jerárquica y lógica.
  • Requiere un controlador de dominio (servidor Windows con AD DS instalado).
  • Permite autenticación centralizada, políticas de seguridad y replicación de datos entre controladores de dominio.

DNS

  • Sistema que traduce nombres de dominio a direcciones IP (A para IPv4, AAAA para IPv6).
  • Esencial para la navegación y localización de servicios en la red.

DHCP

  • Protocolo que asigna automáticamente direcciones IP, máscara de subred, puerta de enlace y DNS a los dispositivos de la red.
  • Simplifica la administración, evitando la configuración manual en cada cliente.

Configuración de Clientes:

Parámetros clave:

  • IP: Dirección única (IPv4 o IPv6) asignada al dispositivo para identificarlo en la red.
  • Submáscara: Define qué parte de la IP corresponde a la red y cuál al host; ejemplo típico en IPv4: 255.255.255.0.
  • Puerto: Número lógico que identifica servicios o aplicaciones en un dispositivo (ejemplo: 80 para HTTP, 443 para HTTPS).
  • Protocolo: Define cómo se comunican cliente y servidor; los más comunes son TCP y UDP, sobre IPv4 o IPv6.
  • MAC: Dirección física única de la tarjeta de red, usada para la comunicación en la capa de enlace de datos.
  • DNS1 y DNS2: Direcciones de los servidores DNS primario y secundario que usa el cliente para resolver nombres de dominio.

Ejemplo configuración:

  • IP: 192.168.1.10 (IPv4) o 2001:db8::10 (IPv6)
  • Submáscara: 255.255.255.0 (IPv4) o /64 (IPv6)
  • Puerto: 80 (HTTP), 443 (HTTPS), etc.
  • Protocolo: TCP/IP
  • MAC: 00:1A:2B:3C:4D:5E
  • DNS1: 8.8.8.8
  • DNS2: 8.8.4.4

Conclusiones:

  • IPv4 y IPv6 son los protocolos principales para el direccionamiento y comunicación en redes, siendo IPv6 la evolución que soluciona las limitaciones de IPv4.
  • Windows, Linux y Mac pueden funcionar como servidores, cada uno con ventajas específicas según el entorno.
  • Active Directory centraliza la gestión de usuarios y recursos en redes Windows, mientras que DNS y DHCP son servicios fundamentales para la asignación y resolución de direcciones.
  • Los clientes requieren parámetros como IP, submáscara, puerto, protocolo, MAC y DNS para conectarse y operar correctamente en la red.
